# Jarvis v5 Flowstate Policy

Flowstate is a first-class lane for handling long-form or source-heavy material.

## Purpose

Flowstate exists to:
- ingest source material
- transcribe or extract useful content
- distill it into reusable artifacts
- propose promotion into more durable system lanes

---

## Core rule

Flowstate is not self-promoting.

A Flowstate output does **not** automatically become:
- a task
- long-term memory
- a system-wide assumption
- a durable operating rule

Promotion requires explicit approval.

---

## Expected artifact path

A Flowstate item should move through a structured path such as:

1. source record
2. transcript / extraction artifact
3. distillation artifact
4. promotion proposal
5. approval decision
6. only then promotion into task/memory/other durable state

---

## Minimum Flowstate records

Each Flowstate source should have:
- source ID
- source type
- intake timestamp
- source location or reference
- processing status
- linked transcript/extraction artifact
- linked distillation artifact
- linked promotion request if created

---

## Promotion questions

Before promotion, the system should be able to answer:
- what is the source?
- what was extracted?
- what was distilled?
- what exact durable object is being proposed?
- why should it be promoted?
- who approved it?

---

## Promotion targets

Approved Flowstate outputs may be promoted into:
- a task
- artifact memory
- working notes
- structured reference material

But only through an explicit reviewed action.

---

## Guardrails

- no silent promotion
- no silent memory writes
- no silent task creation from Flowstate
- no assumption that source material is trustworthy without review
